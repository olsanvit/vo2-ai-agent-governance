PRAVIDLA POUŽITÍ SKILLS

- nezkoušet použít skill který není v tomto seznamu, i kdyby byl dostupný v Claude.ai

## db-schema-guardian
Task types: schema governance, tabulky, sloupce, indexy, BaseGuid, migrace
Description: Autoritativní správce DB schématu. Zajišťuje, že všechny tabulky mají povinné
  sloupce (Guid, CreatedAt, UpdatedAt, Emoji, Colors, scoring sloupce), správné indexy,
  FK ve formátu SomethingGuid a BaseGuid pro multi-type entity. Řídí migrace bezpečně.
Inputs: název tabulky nebo seznam tabulek, požadované schéma změny
Outputs: SQL migration script, schema diff report, bezpečnostní hodnocení migrace
Limitations: nespouští migrace automaticky — vždy vyžaduje potvrzení operátora

## match-upsert-guardian
Task types: identity validace, matching, safe upsert zápasů a entit
Description: Bezpečný upsert s validací identity. Provádí multi-step matching
  (exact name → alias → token 80% → ExternalID → manual queue) před zápisem.
  Chrání před duplikáty a nechtěnými přepisy canonical hodnot.
  Pro sport: validuje HomeTeam/AwayTeam/Competition/Season konzistenci před upsert Matches.
Inputs: entita nebo zápas k upsertnutí s dostupnými identifikátory
Outputs: match result (matched/new/conflict), Guid, upsert výsledek
Limitations: při nejednoznačném matchování posílá do ManualReviewQueue, nezapisuje

## flashscore-match-harvester
Task types: sběr zápasů z Flashscore-like zdrojů, live score pages
Description: Specializovaný harvester pro Flashscore a podobné sportovní výsledkové weby.
  Parsuje výsledky, datum, týmy, skóre, status zápasu. Respektuje rate limiting (1 req/s),
  circuit breaker a per-date limity (max 100 nových zápasů, max 25 dotazů na datum).
Inputs: cílové datum nebo rozsah datumů, sport/competition filter
Outputs: seznam zápasů s HomeTeam, AwayTeam, Score, Status, StartTimeUtc, SourceUrl
Limitations: pouze historické zápasy (včera a starší); dnešní a budoucí PŘESKOČIT

## scheduled-run-report-formatter
Task types: finální run report, formátování výstupu
Description: Generuje strukturovaný run report ve standardním formátu ManagerPrompt.
  Zahrnuje: startup sekci, zpracované zápasy per datum, Post-Match Pipeline výsledky
  (Krok 1-11), errors, highlights, SkillsUsed[], capability_missing[], duration.
Inputs: raw run data (matches processed per date, pipeline results, errors, highlights, SkillsUsed)
Outputs: formátovaný run report připravený pro save_agent_run_report
Limitations: report musí být uložen i při selhání runu

## fallback-run-auditor
Task types: fallback-mode evaluace, degraded run assessment
Description: Vyhodnocuje run v degraded módu (DB nedostupný, zdroje blokované,
  time budget překročen). Rozhoduje, které kroky Post-Match Pipeline lze bezpečně
  odložit (Tier 2/3) a co musí být zachováno jako minimum (Tier 1 + run report).
Inputs: aktuální stav DB, circuit breakerů, zbývající time budget
Outputs: fallback execution plan, odložené kroky s tier klasifikací, doporučení pro next run
Limitations: nelze použít pokud je DB zcela nedostupná (db_unreachable = STOP)

## sheet-fallback-mapper
Task types: export do Google Sheets, fallback při nedostupnosti DB
Description: Exportuje sebraná data zápasů do Google Sheets jako fallback zálohu
  při dočasné nedostupnosti DB. Data jsou strukturována pro pozdější reimport.
Inputs: sebraná data zápasů (Matches, MatchEvents, MatchStats...), cílový spreadsheet ID
Outputs: Google Sheets export s metadaty (timestamp, source, agent, run ID, datum zápasů)
Limitations: pouze fallback — pokud DB dostupná, vždy preferovat přímý zápis do DB

## next-run-calendar-scheduler
Task types: plánování příštího runu v Google Calendar
Description: Vytváří nebo aktualizuje Calendar event pro příští scheduled run v AIZapisy.
  Zajišťuje, že existuje právě JEDNA budoucí událost pro agenta.
  Při úspěšném runu přidá suffix " OK" + počet zpracovaných zápasů, při selhání " X".
Inputs: agent name, výsledek runu, počet zpracovaných zápasů, čas příštího scheduled runu
Outputs: Calendar event (create nebo update), potvrzení
Limitations: nemaže historické události — pouze archivuje duplicitní budoucí

## vo2qnapdb-image-guardian
Task types: image workflow, SharedImages + EntityImages, QNAP upload
Description: Kompletní image workflow: validace (MIME, velikost, hash, bezpečný filename),
  dedup přes SHA256, upload přes save_image_url nebo save_image_base64,
  zápis do SharedImages + EntityImages s historizací (IsCurrent, ValidFrom/ValidTo).
  Pro sport: loga klubů, portréty hráčů/trenérů, venue fotografie.
Inputs: URL nebo base64 obrázku, EntityGuid, EntityType, ImageType
Outputs: SharedImages Guid, EntityImages záznam, upload potvrzení
Limitations: pouze když explicitně vyžadováno; SVG bez scriptů; povolené MIME: png, webp, jpeg, svg+xml
