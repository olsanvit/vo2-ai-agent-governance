=============================================================
CHAPTER 4: GENERATION WORKFLOW
=============================================================

## Priority Logic

1. Pokud všechny TargetTables mají coverage > 95 % → report "Fully covered"
2. Tabulka s nejnižším coverage % = první na řadě
3. Při shodném pokrytí → pořadí dle TargetTables v config.txt
4. Log: "⚖️ Coverage: {Table} {coverage}% → Priority: {winner}"

## Per-Entity Workflow

1. SELECT entity bez obrázku WHERE ScopeFilter LIMIT MaxPerRun
2. Pro každou entitu:
   a. VERIFY: ScopeFilter stále platí + ImageColumn stále prázdný
   b. LOAD DETAIL: načíst všechny atributy entity
   c. BUILD PROMPT: viz Ch 5
   d. GENERATE: create_image(prompt, size=ImageSize)
   e. QUALITY CHECK: viz Ch 6; nesplňuje → 1 retry s upraveným promptem
   f. ASSIGN: upsert_record → ImageColumn = URL obrázku
3. Chyba při generování → skip + error; pokračovat další entitou
4. Chybí data pro rozumný prompt → skip + log "⏭️ Skipped: insufficient data"

=============================================================
CHAPTER 5: PROMPT ENGINEERING
=============================================================

## Prompt Templates

Prompt šablony jsou definovány v {AgentName}_prompts.txt nebo inline v config.txt.
Každý ImageType má vlastní šablonu s placeholdery pro atributy entity.

Příklad struktury šablony:
```
ImageType: logo
Template: "Professional {EntityType} logo for '{Name}'. {Context}. Clean minimalist design. Square format. No text, no letters. 1:1 square."

ImageType: portrait
Template: "Square portrait of {Name}. {Context}. Professional headshot. Confident pose. Realistic style. Chest-up. 1:1 square."
```

## Context Building

Při sestavení promptu:
1. Načíst šablonu pro daný ImageType
2. Doplnit {Name} a další atributy entity dostupné z DB
3. Doplnit {Context} z dostupného kontextu (kategorie, styl, prostředí apod.)
4. Přidat standardní instrukce dle ImageType (viz Quality Rules)

Pokud šablona chybí → sestavit obecný prompt z dostupných atributů.

## Obecná prompt pravidla

Pro loga / ikony (logo ImageType):
- Čtvercový formát 1:1
- Minimalistický styl
- Žádný text, žádná písmena, žádná čísla
- Maximálně 3–5 dominantních barev

Pro portréty / fotky (portrait ImageType):
- Čtvercový formát 1:1
- Realistický styl (ne karikatura, ne anime)
- Jeden subjekt
- Chest-up kompozice

Pro bannery / ilustrace (ostatní typy):
- Čtvercový formát 1:1 (nebo dle ImageSize v config)
- Styl dle specifikace v šabloně

CHAPTER 6: QUALITY & VALIDATION
=============================================================

## Quality Rules (obecné pro všechny ImageTypes)

MUSÍ splňovat (jinak regenerovat s upraveným promptem):
- Čtvercový formát 1:1
- Vizuálně konzistentní styl napříč během
- Výsledek odpovídá požadovanému ImageType

Specifická pravidla per ImageType jsou definována v {AgentName}_prompts.txt.

## Regeneration Protocol

1. Upřesnit prompt (přidat instrukce k problematickému prvku)
2. Vygenerovat znovu (1 retry)
3. Při druhém selhání → skip, zapsat do errors[]

Log: "🔄 Retry: {entityName} — důvod: {failReason}"
Log skip: "⏭️ Skipped after retry: {entityName}"
