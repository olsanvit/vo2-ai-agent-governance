CHAPTER 4: AUDIT WORKFLOW
=============================================================

## Phase 1: Database Table Audit

Použij skill database-table-audit pro:

1. Ověření dostupnosti DB (db_ping)
2. Získání seznamu všech tabulek v databázi:
   run_select_sql("SELECT table_name FROM information_schema.tables WHERE table_schema='public' ORDER BY table_name")
3. Pro každou tabulku:
   - počet řádků: SELECT COUNT(*) FROM {table}
   - poslední aktualizace: SELECT MAX(UpdatedAt) FROM {table} (pokud sloupec existuje)
     nebo MAX(CreatedAt) jako fallback
   - seznam sloupců: SELECT column_name FROM information_schema.columns WHERE table_name='{table}'
4. BaseGuid soulad: ověřit přítomnost Guid, CreatedAt, UpdatedAt, Emoji, Colors
5. Log: "📋 Audit: {N} tabulek načteno"

Pokud tabulka nemá UpdatedAt ani CreatedAt → datum poslední aktualizace je "nedostupné".
Neodhaduj chybějící hodnoty; jasně označ v reportu.

## Phase 2: Safe Schema Patch

Pro každou tabulku kde chybí BaseGuid sloupce:

Podmínky pro bezpečné doplnění:
- Je dostupný schema nástroj (ensure_column nebo ALTER TABLE přes run_select_sql)
- Změna je čistě aditivní (přidání sloupce, nikdy změna nebo mazání)
- Tabulka existuje a je dostupná

Pravidla doplnění:
- Guid uuid → DEFAULT gen_random_uuid(), UNIQUE, NOT NULL
- CreatedAt timestamptz → DEFAULT now(), NOT NULL
- UpdatedAt timestamptz → DEFAULT now(), NOT NULL
- Emoji text → DEFAULT '', NOT NULL
- Colors text → DEFAULT '', NOT NULL

NIKDY:
- Nepřejmenovávat existující Id sloupce
- Neměnit existující datové typy
- Nestahovat hodnoty ze stávajících dat
- Nespouštět bez potvrzení pokud uživatel výslovně nepožádal

Pokud doplnění není možné (nástroj nedostupný, destruktivní riziko) → označit jako schema_gap v reportu.

## Phase 3: Freshness Evaluation

Použij skill table-freshness-report pro kategorizaci každé tabulky:

| Kategorie      | Podmínka                           |
|----------------|------------------------------------|
| V pořádku      | poslední update < 24 hodin         |
| Mírné zpoždění | 24 hodin – 3 dny                   |
| Pozor          | 3 – 7 dní                         |
| Varování       | 8 – 30 dní                        |
| Problém        | > 30 dní nebo datum nedostupné     |

Výchozí předpoklad: denní aktualizace.
Pokud je pro konkrétní tabulku znám jiný interval → zohledni ho.
U každé tabulky uveď krátké vysvětlení zařazení do kategorie.

## Phase 4: New Entity Suggestions

Na konci každého reportu navrhni 10 názvů tabulek, které dávají smysl doplnit.

Pravidla:
- Navrhuj entity, které v DB ještě neexistují
- Vybírej dle aktuální struktury a tematického pokrytí DB
- U každé entity uveď doporučeného primárního agenta/katalog podle obsahu (ne podle jména tabulky)
- Pokud je zařazení hraniční, uveď 1–2 sekundární alternativy
- Nenavrhuj duplikáty stávajících tabulek

Formát návrhu:
- **NázevTabulky** — krátký popis (1 řádek) — Primární agent: {AgentName}

CHAPTER 5: BASEGUID STANDARD
=============================================================

## Povinné sloupce

Každá tabulka MUSÍ mít:

| Sloupec   | PostgreSQL typ                  | Výchozí hodnota      | Pravidlo                          |
|-----------|---------------------------------|----------------------|-----------------------------------|
| Guid      | uuid                            | gen_random_uuid()    | UNIQUE, NOT NULL; po vytvoření neměnit |
| CreatedAt | timestamp with time zone        | now()                | NOT NULL; nastavit při insertu, neměnit |
| UpdatedAt | timestamp with time zone        | now()                | NOT NULL; aktualizovat při každém UPDATE |
| Emoji     | text                            | ''                   | NOT NULL; UI označení             |
| Colors    | text                            | ''                   | NOT NULL; UI barvy                |

## Insert pravidla

- Guid: pokud nedodán → vygenerovat (gen_random_uuid())
- CreatedAt: pokud nedodán → aktuální UTC čas
- UpdatedAt: pokud nedodán → aktuální UTC čas
- Emoji: pokud nedodáno → prázdný string
- Colors: pokud nedodáno → prázdný string

## Update pravidla

- Guid → nikdy neměnit
- CreatedAt → nikdy neměnit (výjimka: výslovný bezpečný důvod)
- UpdatedAt → vždy aktualizovat na aktuální UTC čas
- Emoji, Colors → měnit pouze pokud je to součást požadované změny nebo UI normalizace

## Scoring sloupce (volitelné, doporučené)

Pokud tabulka uchovává hodnotitelná data, přidat 22 scoring sloupců numeric(6,3):
FinalScore, DataScore, QualityScore, ConfidenceScore, ReliabilityScore, CoverageScore,
HistoricalScore, ValidationScore, a dalších 14 dle domény.
