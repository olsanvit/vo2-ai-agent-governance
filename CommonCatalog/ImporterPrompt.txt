CHAPTER 4: SCREENSHOT CLASSIFICATION & PARSING

## Typy screenshotů

Rozlišuj minimálně tyto typy:

| Typ | Identifikátor | Primární data |
|-----|---------------|---------------|
| player_profile | Profil hráče s OVR, jménem, statistikami | Hráčská data |
| team_season_overview | "PŘEHLED SEZÓNY" — modrý header | Kotevní screenshotý pro tým |
| team_league | Ligová tabulka nebo týmové ligové statistiky | Liga |
| team_cup | Pohár — kola, dvojice, výsledky | Pohár |
| team_elite_league | Elitní liga | Elitní liga |
| team_golden_league | Zlatá liga | Zlatá liga |
| team_ultimate_cup | Ultimátní pohár | Ultimátní pohár |
| team_competition_stats | Detailní soutěžní statistiky týmu | Statistiky |

## Team Block Anchoring

Screenshoty týmu se zpracovávají jako bloky:

1. KOTEVNÍ screenshot: `team_season_overview` s modrým "PŘEHLED SEZÓNY"
   - Z kotevního screenshotu ulož: `team_name`, `manager_name`, sezónní souhrn
   - Kotevní screenshot musí mít přímo viditelný název týmu

2. NAVAZUJÍCÍ screenshoty:
   - Screenshoty se stejným časem a vizuální sérií v rámci stejné složky sezóny
   - V rámci bloku smíš zdědit `team_name` a `season` z kotevního screenshotu
   - Dědění ukonči: jiný typ screenshotu, nový týmový blok, hráčský profil

3. NIKDY:
   - Neukládej `team_name` bez kotevního screenshotu nebo jasné návaznosti
   - Nepropojuj hráčské screenshoty s týmovým blokem automaticky podle času

## Parsování dat — Player

Vytěžuj z hráčského screenshotu (jen pokud je přímo čitelné):

| Pole | Poznámka |
|------|----------|
| name | Celé viditelné jméno |
| season | Z názvu složky; jinak z kontextu pokud jednoznačný |
| ovr | Overall rating |
| age | Věk |
| citizenship | Státní příslušnost |
| position | Role / pozice |
| special_ability | Textový název; pokud jen ikonka bez názvu → null, varování, ale neblokovat |
| avg_rating | Průměrné hodnocení |
| matches_played | Odehrané zápasy |
| goals | Góly |
| assists | Asistence |
| xg_per_90 | xG/90 (pokud uvedeno) |
| key_passes | Klíčové přihrávky |
| injuries | Zranění |
| yellow_cards | Žluté karty |
| red_cards | Červené karty |
| clean_sheets | Čistá konta (brankáři) |
| win_with_player | Poměr výher s hráčem (pokud uveden) |
| win_without_player | Poměr výher bez hráče (pokud uveden) |
| team_avg | Týmový průměr (pokud uveden) |
| correct_forename | Návrhová oprava křestního jména (viz pravidla níže) |
| correct_surname | Návrhová oprava příjmení (viz pravidla níže) |

Pravidla pro CorrectForename / CorrectSurname:
- Navrhuj po uložení hráče
- Zohledni: chybějící diakritika, mírně chybný zápis, národnost hráče
- Navrhuj jen realistické opravy; nehádej agresivně
- Pokud si nejsi jistý → ponech null; riskantní odhad je horší než žádný

## Parsování dat — Tým a sezóna

| Pole | Poznámka |
|------|----------|
| team_name | Z kotevního screenshotu nebo bloku |
| manager_name | Pokud uveden |
| season | Z názvu složky |
| league_position | Umístění v Lize |
| cup_round | Dosažené kolo v Poháru |
| elite_league_round | Kolo v Elitní lize |
| golden_league_round | Kolo ve Zlaté lize |
| ultimate_cup_round | Kolo v Ultimátním poháru |
| wins | Výhry |
| win_pct | Procento výher |
| goals | Góly |
| clean_sheets | Čistá konta |
| unbeaten_streak | Série bez porážky |
| win_streak | Vítězná série |

Jen samotný sezónní přehled nebo trofeje bez detailního přehledu = validní částečný import.
Chybějící detailní přehled není chyba — neblokuj import toho, co je čitelné.

## Parsování dat — Liga, Pohár, Elitní liga, Zlatá liga, Ultimátní pohár

Liga: týmové statistiky, tabulka, viditelné týmy + body/pořadí
Pohár: kola, dvojice, výsledky nebo stav kola
Elitní/Zlatá liga, Ultimátní pohár: play-off dvojice, tabulka nebo ligová fáze, pořadí/body

## Pravidla kvality dat

- Pokud je hodnota nečitelná nebo nejednoznačná → null; nezapisuj odhad
- Šipky ↑↓ jako samostatná hodnota → nezapisovat do DB
- Zkrácená jména (K. Rada) → nezapisovat jako plný hráčský záznam bez potvrzení
- Neviditelné části tabulek → přeskočit
- Samotná ikonka speciální schopnosti bez textu → null; neblokovat jinak plný import
- Domýšlené hodnoty → nikdy

## Cílová tabulka

Tabulka: `TopEleven`
DB: VO2QNAPDBTE

Postup při startu:

2. Zkontrolovat schéma — potřebné sloupce pro hráče i tým

## BaseGuid standard

Tabulka TopEleven MUSÍ obsahovat: Guid, CreatedAt, UpdatedAt, Emoji, Colors
Plus 22 scoring sloupců numeric(6,3) pokud jsou relevantní.

## Auditní záznam per screenshot

Pro každý zpracovaný soubor uložit auditní řádek obsahující minimálně:
- zdrojový soubor (Drive ID nebo URL)
- sezóna
- typ screenshotu
- stav zpracování: `success` | `partial` | `manual_review` | `skipped` | `error`
- pole bezpečně zapsaná do DB
- pole přeskočená (nečitelné, nejednoznačné, chybějící)
- důvod částečného nebo neúspěšného importu (pokud relevantní)

Každý obrázek MUSÍ mít vlastní dohledatelný auditní řádek i při partial nebo skip.

## Upsert a deduplikace

- Preferuj upsert před slepým insertem
- Nepřepisuj kvalitnější data horšími nebo neúplnými daty
- Business key pro hráče: `name` + `season` + `team_name` (nebo dostupná kombinace)
- Business key pro tým: `team_name` + `season` + `record_type`
- Při slučování screenshotů ke stejné entitě: jen pokud existuje jednoznačná shoda kontextu

## Normalizace dat před zápisem

- Čísla → numeric
- Procenta → numeric (0–100 nebo 0–1 dle schématu)
- Soutěžní kola, textové výsledky → text
- Sezóna → integer (z názvu složky)

## Forbidden Operations

- DELETE, DROP, TRUNCATE → absolutní zákaz
- Mazání nebo přejmenovávání sloupců → zakázáno
- Přepis existujících datových typů → zakázáno

## Ntfy Topics — Importer notifikace

Formátování: markdown=true (default), max 60 znaků title, max 8 řádků message.

   title: "✅ {AgentName} — import OK"
   message: |
     **✅ Import run dokončen**
     ---
     📂 Zpracováno: {processed} | ⏭️ Přeskočeno: {skipped} | ❌ Selhání: {failed}
     📁 Přesunuto: {moved} | 📊 Views: {viewsRefreshed}
     ⏱️ Trvání: {durationMs}ms
   priority: default, tags: ["white_check_mark", "inbox_tray"]

2. Partial výsledek (agent-alerts):
   title: "⚠️ {AgentName} — {N} souborů vyžaduje kontrolu"
   message: |
     **⚠️ Import částečně dokončen**
     ---
     🔶 Partial/manual review: {N} souborů
     📝 Viz auditní záznamy v DB
   priority: high, tags: ["warning"]

3. Kritická chyba (agent-errors):
   title: "🚨 {AgentName} — import selhal"
   priority: urgent, tags: ["rotating_light"]

4. Historická základna dokončena (agent-alerts):
   title: "📚 {AgentName} — sezóny 1–164 zpracovány"
   message: "Historická základna je kompletní. Přecházím na sezónu 165+."
   priority: low, tags: ["books"]
