-- A. AUDIT CORE (2 — POVINNÉ) --------------------------------

## database-table-audit
Task types: audit databáze, seznam tabulek, počty řádků, metadata
Description: Hlavní audit skill. Ověří dostupnost DB, načte seznam všech tabulek
  v public schema, pro každou tabulku zjistí počet řádků, datum poslední aktualizace
  (MAX(UpdatedAt) nebo MAX(CreatedAt) jako fallback) a seznam sloupců.
  Vrátí strukturovaný přehled pro další zpracování.
Inputs: DB připojení, volitelně: seznam konkrétních tabulek nebo schema filtr
Outputs: seznam tabulek s {name, rowCount, lastUpdated, columns[], available}
Limitations: MAX(UpdatedAt) není dostupný pokud tabulka sloupec nemá; fallback na CreatedAt;
  pokud ani to → lastUpdated="nedostupné"

## table-freshness-report
Task types: freshness analýza, kategorizace, finální report
Description: Vyhodnotí stáří dat pro každou tabulku dle kategorií z CheckerPrompt Ch 4.
  Přiřadí kategorii (V pořádku / Mírné zpoždění / Pozor / Varování / Problém),
  připraví krátké vysvětlení pro každou tabulku.
  Sestaví finální report ve formátu z CheckerPrompt Ch 6.
Inputs: výstup z database-table-audit, aktuální čas, volitelná konfigurace intervalů
Outputs: kompletní report (Souhrn + Detail tabulek + Bezpečnostní potvrzení)
Limitations: výchozí interval = 1 den; přesnější intervaly jen pokud jsou explicitně known

-- B. BASEGUID COMPLIANCE (3) ----------------------------------

## baseguid-compliance-checker
Task types: BaseGuid standard, kontrola povinných sloupců
Description: Pro každou tabulku ověří přítomnost všech 5 BaseGuid sloupců:
  Guid (uuid), CreatedAt (timestamptz), UpdatedAt (timestamptz), Emoji (text), Colors (text).
  Hlásí chybějící sloupce per tabulka. Určí celkový compliance % napříč DB.
Inputs: seznam tabulek se sloupci (z database-table-audit)
Outputs: per-table compliance report {tableName, compliant, missingColumns[]}
Limitations: pouze přítomnost sloupců; neověřuje správnost datových typů automaticky

## safe-schema-patcher
Task types: bezpečné přidání chybějících BaseGuid sloupců
Description: Přidá chybějící BaseGuid sloupce bezpečnou aditivní ALTER TABLE operací.
  Podmínky: dostupný schema nástroj, čistě aditivní změna, tabulka existuje.
  Výchozí hodnoty: Guid→gen_random_uuid(), CreatedAt/UpdatedAt→now(), Emoji/Colors→''.
  NIKDY neodstraňuje existující sloupce, nemění datové typy, nepřejmenovává Id.
Inputs: tabulka, seznam chybějících sloupců, konfirmace od uživatele nebo explicit_safe_patch=true
Outputs: přidané sloupce[], schema_gap pokud nelze provést
Limitations: vyžaduje confirm pokud uživatel výslovně nepožádal; odmítne jakoukoliv destruktivní operaci

## schema-gap-reporter
Task types: schema gap, hlášení nesplnitelných oprav
Description: Zaznamenává tabulky kde BaseGuid sloupce chybí a doplnění nebylo možné
  (nástroj nedostupný, riziko destrukce, uživatel nepožádal).
  Generuje jasné schema_gap záznamy do reportu.
Inputs: seznam tabulek s compliance výsledky, výsledky safe-schema-patcher
Outputs: seznam schema_gap záznamů {tableName, missingColumns[], reason}
Limitations: pouze reportovací skill; neprovádí žádné DB operace

-- C. ANALYSIS (3) ---------------------------------------------

## db-null-audit
Task types: NULL analýza, datová kvalita, prioritizace
Description: Skenuje tabulky a reportuje NULL hodnoty per klíčový sloupec.
  Prioritizuje sloupce: Guid, UpdatedAt, CreatedAt (kritické), pak ostatní.
Inputs: seznam tabulek nebo ALL
Outputs: NULL audit report (tabulka, sloupec, NULL ratio, priority rank)
Limitations: max 200 tabulek na run; aproximativní NULL count

## cross-db-consistency-checker
Task types: konzistence přes více DB, entity existence check
Description: Ověří konzistenci entit přes více MCP připojení.
  Typický případ: entita odkazovaná v VO2QNAPDBAI existuje v VO2QNAPDBUSM?
  Vrátí seznam broken references.
Inputs: zdrojová DB, cílová DB, tabulky k porovnání, join key
Outputs: broken references[], orphan records[], consistency score
Limitations: pouze pro přímé FK/reference; neprovádí deep-merge

## row-count-anomaly-detector
Task types: anomálie v počtech řádků, neočekávaný pokles/nárůst
Description: Porovná aktuální počty řádků s historickými hodnotami z AgentRunReports.
  Upozorní na neočekávané poklesy (možná nechtěné mazání) nebo velké nárůsty.
  Threshold: pokles > 5% nebo nárůst > 200% oproti poslednímu auditu.
Inputs: aktuální rowCounts, historické runReports
Outputs: anomálie[], normální tabulky[]
Limitations: min 2 předchozí audity pro smysluplné porovnání

## db-table-ownership-checker
Task types: ownership, permissions, AgentAI, tabulky
Description: Ověřuje, že všechny tabulky v public schema vlastní AgentAI. Detekuje tabulky s jiným ownerem. Navrhuje ALTER TABLE X OWNER TO "AgentAI". Spouštět po ensure_table a při startup DB checku.
Inputs: seznam tabulek nebo ALL
Outputs: ownership report per tabulka (owner: AgentAI/other), SQL fix script pro tabulky s jiným ownerem
Limitations: ALTER TABLE nespouští automaticky — vyžaduje superuser potvrzení nebo roundnet uživatele

## mcp-version-validator
Task types: MCP version check, startup, minimum version, mcp_version_mismatch
Description: Ověřuje MCP_VERSION >= 10.0.18 při startupu. Pokud verze je nižší → hlásí mcp_version_mismatch v AgentRunReport, přeskočí všechny mutační operace (upsert, ensure_table, ensure_columns), pokračuje pouze v read-only módu.
Inputs: current MCP_VERSION (string), minimum required version (default "10.0.18")
Outputs: pass/fail, action_taken (proceed/skip_mutations), version_delta
Limitations: porovnává semver major.minor.patch; pre-release tagy ignoruje

## schema-mixed-case-detector
Task types: PascalCase vs lowercase, duplicitní sloupce, schema cleanup, schema_mixed_case_detected
Description: Detekuje tabulky kde existují duplicitní sloupce vzniklé smíšením starého lowercase a nového PascalCase schématu (např. "name" i "Name" ve stejné tabulce). Navrhuje DDL pro odstranění starší varianty. Reportuje jako schema_mixed_case_detected.
Inputs: tabulka nebo ALL, preferred_case: PascalCase/lowercase (default PascalCase)
Outputs: seznam tabulek s konfliktními sloupci, doporučený DROP COLUMN skript, rizikové záznamy (kde starý sloupec má data)
Limitations: DROP nespouští automaticky — data risk; při nejasnosti posílá do ManualReviewQueue

## mcp-502-recovery-handler
Task types: 502 error, MCP nedostupnost, retry logika, db_502_permanent
Description: Zpracovává 502 Bad Gateway errory z MCP serveru. Exponential backoff: 1s → 2s → 4s, max 3 pokusy. Po 3 selháních nastaví db_502_permanent, přeskočí všechny DB operace pro daný run, pokračuje s Drive/API tasky. Loguje každý retry pokus.
Inputs: failed tool call, retry_count (aktuální), max_retries (default 3)
Outputs: retry result (success/fail), skip_decision (proceed/skip_db), total_wait_ms
Limitations: pouze pro 502 HTTP status; jiné chyby (404, 500, timeout) neřeší tímto flow

## governance-prompt-cache-sync
Task types: AgentPromptCache, verze synchronizace, cache invalidace, cache_refreshed
Description: Po načtení nové verze governance promptu z Drive nebo Gitea aktualizuje záznam v AgentPromptCache tabulce. Ověřuje, že cached PromptVersion odpovídá právě načtené. Pokud ne → přepíše cache novým obsahem a loguje cache_refreshed v Metadata.
Inputs: agent_name, prompt_content, new_version, source (gitea/drive/github)
Outputs: cache_status (hit/miss/refreshed), previous_version, new_version
Limitations: cache write může selhat pokud DB nedostupná — v takovém případě přeskoč cache update a pokračuj

## agent-startup-health-broadcaster
Task types: ntfy, startup summary, readiness report, agent-status
Description: Po dokončení startup sekvence pošle na ntfy kanál agent-status stručné shrnutí: agent name, prompt verze, MCP verze, DB status (ok/unavailable), počet dostupných tabulek, prompt source (gitea/drive/cache), run ID. Pomáhá operátorovi sledovat stav všech agentů po jejich spuštění.
Inputs: všechny výsledky startup sekvence (version, db_status, tables_count, prompt_source, run_id)
Outputs: ntfy zpráva odeslaná na agent-status topic, startup_summary objekt pro AgentRunReport
Limitations: pokud ntfy nedostupné → přeskoč, loguj ntfy_unavailable, nepřerušuj startup

## skill-changelog-recorder
Task types: audit skills, changelog, verzování, skill_added/skill_modified/skill_removed
Description: Loguje přidání, úpravu nebo odebrání skills. Záznam obsahuje: datum, skill_name, change_type (added/modified/removed), reason, operator. Záznamy ukládá do sekce SKILL_CHANGELOG na konci CheckerPromptSkills.txt a do AgentRunReport jako metadata.
Inputs: skill_name, change_type, reason, operator (volitelně)
Outputs: changelog entry s datumem, status (logged/error)
Limitations: pouze loguje — nespouští žádné akce automaticky
