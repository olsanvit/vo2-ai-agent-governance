## bot-coverage-analyzer
Task types: coverage check, bot entity statistics, image coverage %
Description: Analyzuje pokrytí obrázků pro bot entity v USM.
  Spočítá: celkový počet bot týmů, počet bez loga, coverage %.
  Totéž pro bot hráče. Vrátí porovnání a doporučení priority.
Inputs: DB připojení, název tabulek (Teams, Players nebo ekvivalenty)
Outputs: teamsCoverage %, playersCoverage %, priority doporučení
Limitations: pracuje pouze s IsBot = true entitami

## bot-entity-verifier
Task types: verify IsBot, image missing check, pre-generate validation
Description: Před každým generováním znovu ověří, že entita splňuje scope:
  IsBot = true a obrázek skutečně stále chybí (NULL nebo prázdný).
  Chrání před zápisem na nesprávnou entitu.
Inputs: EntityGuid nebo jiný identifikátor, tabulka
Outputs: verified/skip flag, skip reason pokud odmítnut
Limitations: přímý DB dotaz; nepoužívat cached data ze startup fáze

## missing-image-batch-loader
Task types: načtení dávky entit bez obrázku, max 10
Description: Načte seznam max 10 bot entit bez obrázku (LogoUrl nebo PhotoUrl NULL/prázdný).
  Řadí dle data vytvoření (nejstarší první).
  Respektuje scope: IsBot = true only.
Inputs: tabulka, sloupec s URL obrázku, max batch size (default 10)
Outputs: seznam entit s jejich Guid a dostupnými atributy
Limitations: max 10 entit per volání; neřadí dle priority scoringu

  Zajistí "chest-up, one person, realistic" instrukci.
Inputs: detail hráče (FullName, Position, BotArchetype, sport context)
Outputs: hotový prompt string pro create_image

## sport-context-resolver
Task types: detekce sportu z Position nebo BotArchetype
Description: Rozpozná sport entity z dostupných atributů (Position, BotArchetype, TeamName).
  Mapuje na sport-specific prompt rozšíření dle tabulky v Ch 5 GeneratorPrompt.
Inputs: Position (string), BotArchetype (string, volitelně), TeamName (string, volitelně)
Outputs: detekovaný sport, prompt extension string, confidence (high/medium/low)
Limitations: nejistá detekce → confidence=low, použít generic "sport background"

## prompt-refinement-handler
Task types: upřesnění promptu po quality check fail, retry
Description: Pokud vygenerovaný obrázek nesplňuje quality rules, upřesní prompt
  přidáním konkrétních instrukcí k problematickým prvkům.
  Loga s textem → přidat "Strictly no text, no letters, no numbers".
  Portrét s více osobami → přidat "Only one athlete, single person".
Inputs: původní prompt, důvod quality fail
Outputs: upřesněný prompt pro 1 retry
Limitations: max 1 refinement per entita; při druhém selhání → skip
