## top-eleven-screenshot-parser
Task types: klasifikace screenshotu, OCR vytěžení, normalizace dat
Description: Domain-specifický skill (Top Eleven). Pro jiné domény vytvořit ekvivalentní parser. Hlavní parsing skill. Klasifikuje screenshot (player_profile,
  team_season_overview, team_league, team_cup, elite/golden/ultimate, competition_stats).
  Vytěží strukturovaná data z jednoho nebo více obrázků. Vrátí normalizovaný
  datový objekt s jistotou per pole a seznam polí kde hodnota nebyla čitelná.
  Pro hráče navrhne CorrectForename/CorrectSurname.
Inputs: obrázek (Drive URL nebo base64), typ screenshotu (pokud znám), sezóna z kontextu
Outputs: {type, season, data{}, confidencePerField{}, skippedFields[], warnings[]}
Limitations: speciální schopnost z ikonky bez textu → null; nezapisovat odhad

## screenshot-type-classifier
Task types: určení typu screenshotu bez plného parsování
Description: Rychlá klasifikace screenshotu do jednoho z kanonických typů
  (player_profile, team_season_overview, team_league, team_cup, team_elite_league,
  team_golden_league, team_ultimate_cup, team_competition_stats).
  Používá vizuální klíče (modrý header, layout, viditelné texty).
Inputs: obrázek (Drive URL nebo base64)
Outputs: screenshot_type, confidence (high/medium/low), visual_cues[]
Limitations: low confidence → doporučit manuální review; neblokovat parsování

## team-block-anchor-detector
Task types: detekce kotevního screenshotu, team block management
Description: Identifikuje kotevní team_season_overview screenshot (modrý "PŘEHLED SEZÓNY",
  viditelný team_name). Spravuje kontext bloku: dědí team_name a season pro navazující
  screenshoty; ukončuje dědění při změně typu nebo novém bloku.
Inputs: screenshot_type, visual_cues, čas souboru, pořadí v rámci složky
Outputs: is_anchor, block_context{team_name, season}, inherit_from_anchor flag
Limitations: bez kotevního screenshotu → team_name null; nededit přes hráčský profil

## name-correction-suggester
Task types: CorrectForename, CorrectSurname, oprava diakritiky
Description: Po uložení hráče navrhne opravy CorrectForename a CorrectSurname.
  Zohledňuje: chybějící diakritika, mírný překlep, národnost hráče.
  Navrhuje jen realistické opravy; pokud nejistý → null.
Inputs: jméno hráče, citizenship/nationality (pokud dostupná)
Outputs: correct_forename (string nebo null), correct_surname (string nebo null), confidence
Limitations: nízká jistota → null; riskantní odhad je horší než prázdné pole

## mcp-502-recovery-handler
Task types: 502 error, MCP nedostupnost, retry logika, db_502_permanent
Description: Zpracovává 502 Bad Gateway errory z MCP serveru. Exponential backoff: 1s → 2s → 4s, max 3 pokusy. Po 3 selháních nastaví db_502_permanent, přeskočí všechny DB operace pro daný run, pokračuje s Drive/API tasky. Loguje každý retry pokus.
Inputs: failed tool call, retry_count (aktuální), max_retries (default 3)
Outputs: retry result (success/fail), skip_decision (proceed/skip_db), total_wait_ms
Limitations: pouze pro 502 HTTP status; jiné chyby (404, 500, timeout) neřeší tímto flow
